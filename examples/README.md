# Examples

This folder contains example designs to demonstrate the functionality of the UPEC Tool.

- **AES-T200**: A cryptographic accelerator infected with an hardware trojan.
                We show how to utilize the UPEC Tool to identify the malicious component.
- **Binary GCD**: An arithmetic unit that computes greatest common divisor of the two operands.
                  We utilize the UPEC-DIT functionality to detect data-dependent timing.
- **HMAC**: A module for message authentication.
            This example demonstrates how to use the UPEC tool to verify the absence of data-dependent timing side channels and hardware trojans.
- **SHA 512**: A cryptographic accelerator.
               The UPEC Tool verifies the absence of data-dependent timing channels through structural analysis.
